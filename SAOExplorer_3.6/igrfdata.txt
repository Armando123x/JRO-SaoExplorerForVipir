IGRFData V3.0
;  Format versions 2.0 and 3.0 are the same, 3.0 is recommended to put
;  here because of early version software hack
;
;  File structure
;  1) This the text file.
;  2) If line starts with semicolon(;) or sharp(#) symbols or line consist
;     of only spaces then it considered as comment line otherwice it 
;     considered as data line (exclusion 1st line)
;  3) First line should start with "IGRFData "
;  4) All subsequent lines are either comment or data lines
;  5) All data consists of several data arrays
;  6) Data array occupies one line or several data lines;
;  7) Elements of array are separated by comma
;  8) If last element on the data line is followed by comma then continuation
;     of this array will be on the next data line
;  9) First element of each array is Date/Time and the next 210 ( 105 G-coeff 
;     and then 105 H-coeff) are the Legendre coefficients for given Date/time
;     (Last array has slightly different meaning)
; 10) Date/Time can be presented using one of the following forms:
;           yyyy
;           yyyy/MM
;           yyyy/MM/dd
;           yyyy/MM/dd (DDD)
;           yyyy/MM/dd (DDD) HH
;           yyyy/MM/dd HH
;           yyyy/MM/dd (DDD) HH:mm
;           yyyy/MM/dd HH:mm
;           yyyy/MM/dd (DDD) HH:mm:ss
;           yyyy/MM/dd HH:mm:ss
;
;     instead slash (/) symbol you can use also dot (.) or dash(-) symbols.
;
;     and <G10>, <G11> and <H11> are geodipole moment coefficient that
;     correspond given datetime (for last data line it have a little
;     different meaning, see later)
; 11) All data arrays should be given in ascending time order.
; 12) Last data array has slightly different meaning:
;     Date/Time is the time from which it is not recommended to use
;     IGRF.
;     First 45 G delta-values and first 45 H delta-values are linear 
;     extrapolation delta per year for G- and H-coeff beginning from
;     previous array. Discarding the changes of the rest 105 - 45 = 60
;     G and H coefficients.
